9 ## netCDF-CXX4 v4.3.1 released TBD
10 
11 * Corrected an issue where cmake-based builds weren't generating `ncxx4-config`. See [GitHub pull request #37](https://github.com/Unidata/netcdf-cxx4/pull/37) for more information.

13 ## netcdf-cxx4 v4.3.0 released May 13, 2016
14 
15 * Fixed an issue where the tests were failing silently, and the underlying `NcType` class could not properly determine the type name or type size reliably. See [GitHub issue #30](https://github.com/Unidata/netcdf-cxx4/issues/30) for more information.
16 * Changed `NCXX_ENABLE_DOXYGEN` option to an easier-to-remember `ENABLE_DOXYGEN`.
17 * Added `--enable-doxygen`, `-DNCXX_ENABLE_DOXYGEN=ON` options to allow generation of netCDF-CXX4 documentation via doxygen using either `configure` or `cmake`, respectively.
18 * Added `netcdf-cxx4` to the [Coverity Scan Dashboard](https://scan.coverity.com/projects/unidata-netcdf-cxx4?tab=overview).
19 * Added `open` and `close` methods for NcFile. See [Github Pull Request #18](https://github.com/Unidata/netcdf-cxx4/pull/18) for more information.
20 * Added `netcdf-cxx4` to travis-ci.org. See [https://travis-ci.org/Unidata/netcdf-cxx4](https://travis-ci.org/Unidata/netcdf-cxx4) for more details.
21 * Added `NcCompoundType` methods `getMemberName` and `getMemberIndex`. See [Pull Request #19](https://github.com/Unidata/netcdf-cxx4/pull/19) for more details.
22 * Added `cmake` support to `netcdf-cxx4`, which will allow us to create a `CDash` continuous integration dashboard similar to those created for the `netcdf-c` and `netcdf-fortran` projects.
23 * Added a `travis-ci` configuration file, `.travis.yml`.
24 * Created `RELEASE_NOTES.md`.
